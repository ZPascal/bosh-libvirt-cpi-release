Find the LXC section (search for `## LXC` or `### LXC`) and replace its content with:

```markdown
## LXC (Linux Containers)

LXC containers are managed via `lxc:///` (privileged) or `lxc+unix:///` (session).

### Prerequisites

```bash
sudo apt-get install -y lxc libvirt-daemon-driver-lxc libvirt-dev
sudo systemctl enable --now libvirtd

# Grant your user access to the libvirt socket
sudo usermod -aG libvirt $USER
```

### Configuration

```json
{
  "BackendURI": "lxc:///",
  "StoreDir": "~/.bosh_libvirt_cpi",
  "Agent": { ... }
}
```

### Known limitations

- LXC containers share the host kernel. BOSH stemcells must be built for LXC
  (raw disk image format, no kernel inside the stemcell).
- The domain builder uses `type='lxc'` with filesystem mounts — not block devices.
  This means `CreateStorageVol` is unused for LXC; disk images are raw files on the filesystem.
- Privileged containers (`lxc:///`) require root. For unprivileged containers,
  configure cgroups delegation and use `lxc+unix:///` with `libvirtd` running as the target user.
- Live migration is not supported for LXC domains.
```

Find the VirtualBox section and replace with:

```markdown
## VirtualBox

VirtualBox VMs are managed via `vbox:///session` (per-user session, unprivileged).

### Prerequisites

```bash
# Install VirtualBox
sudo apt-get install -y virtualbox

# Install the libvirt VBox driver
sudo apt-get install -y libvirt-daemon-driver-vbox libvirt-dev

sudo systemctl enable --now libvirtd
sudo usermod -aG libvirt $USER
```

### Configuration

```json
{
  "BackendURI": "vbox:///session",
  "StoreDir": "~/.bosh_libvirt_cpi",
  "Agent": { ... }
}
```

### Known limitations

- `vbox:///session` connects to the per-user VirtualBox daemon. Each OS user has
  an independent set of VMs — do not mix `vbox:///session` and `vbox:///system`.
- The `libvirt-daemon-driver-vbox` package must be installed and the `vboxdrv`
  kernel module must be loaded (`modprobe vboxdrv`).
- Disk images use VMDK format. Stemcells must provide a `image.vmdk` file.
- Snapshots are not supported by this CPI implementation.
- VirtualBox does not support live migration; `RebootVM` triggers a hard reset.
```
